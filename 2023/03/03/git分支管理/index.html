​ 5.bug分支

​ 软件开发中，bug需要修复，在git中由于分支如此强大，所以 每个bug都可以通过一个新的临时分支来修复，修复后后，合并分支，然后将临时分支删除

​ (1)当接到一个修复带好001的bug时，你要创建一个分支bug—001来修复它，但当前正在dev分支上进行工作还没有提交

​

​ (2)git还提供了一个atash功能，可以把当前工作现场”储存”起来，等以后回复现场后继续工作

​

​ 这个时候git status就会显示工作区干净

​ （3)首先确定在乃个分支上修复bug，嘉定需要在master分支上修复，就从master上创建创建临时分支

​

​ (4)现在修复bug，把the new line删掉，然后提交[git add code.txt git commit -m “修复bug-001”]

​

​ (5)修复完成之后，切换到maser分支，并完成合并，最后删除bug-001分支
